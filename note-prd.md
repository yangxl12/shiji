# 本地笔记 PWA ——「拾记」
**文档版本**：v1.0  
**产品形态**：纯本地 PWA（Progressive Web App）  
**目标平台**：小米 Android 手机

**数据存储**：设备本地 IndexedDB，无任何网络请求  
---
## 一、产品概述
### 1.1 产品定位
一款**零联网、零注册、零服务器**的极简本地笔记工具。打开即用，数据完全驻留在用户设备上。追求「纯粹、克制、高级」的视觉质感与符合人机工程学的交互体验。
### 1.2 核心价值
- **零摩擦**：无登录、无网络依赖、无广告、无权限索取（除必要的存储）
- **直觉化**：分类清晰、操作路径短、符合手指触控习惯
- **安全感**：数据不出设备，用户完全掌控
### 1.3 安装与分发方案

打包为 APK 文件，直接安装到小米手机，无需浏览器

## 二、信息架构
### 2.1 页面层级
```
┌─────────────────────────────────────┐
│           底部 Tab 导航栏            │
│  [随想] [学习] [待办] [标签]         │
└─────────────────────────────────────┘
        │              │
        ▼              ▼
   笔记列表页      标签二级页
   (按时间倒序)    [红][橙][黄][灰]
        │              │
        ▼              ▼
   笔记详情页  ◄──── 同一个详情页组件
   (查看/编辑)
        │
        ▼
   批量操作模式
   (多选 → 批量删除)
```
### 2.2 分类体系定义
**核心原则：分类是「视图」而非「容器」。一条笔记有且仅有一个归属分类（随想/学习/待办），同时可以有零或一个颜色标签（红/橙/黄/灰）。**
| Tab  | 类型     | 可新建 | 说明                                     |
| ---- | -------- | ------ | ---------------------------------------- |
| 随想 | 归属分类 | ✅      | 记录碎片想法，新建时默认归属             |
| 学习 | 归属分类 | ✅      | 记录学习笔记，新建时默认归属             |
| 待办 | 归属分类 | ✅      | 记录待办事项，新建时默认归属             |
| 标签 | 筛选视图 | ❌      | 不允许在此新建笔记，仅按颜色标签筛选显示 |
### 2.3 标签子分类
| 颜色 | 色值（参考） |      |
| ---- | ------------ | ---- |
| 红   | `#E54D42`    |      |
| 橙   | `#F37B1D`    |      |
| 黄   | `#FBBF24`    |      |
| 灰   | `#8C8C8C`    |      |
> 标签页内部以水平排列的 4 个色块 Chip 呈现，点击切换筛选，默认不选中任何颜色时显示全部有标签的笔记（即至少被标记了任一颜色的笔记）。

## 三、数据模型
### 3.1 笔记实体（Note）
```javascript
{
  id: string,          // 主键，crypto.randomUUID() 生成
  title: string,       // 笔记标题，最大 100 字符
  content: string,     // 笔记正文，纯文本，最大 50,000 字符
  category: string,    // 归属分类，枚举值："impromptu" | "study" | "todo"
  tagColor: string | null,  // 颜色标签，枚举值："red" | "orange" | "yellow" | "gray" | null
  createdAt: number,   // 创建时间戳
  updatedAt: number,   // 最后修改时间戳
  isDeleted: boolean   // 软删除标记，默认 false
}
```
### 3.2 IndexedDB 设计
| 项目         | 值                                                                                       |
| ------------ | ---------------------------------------------------------------------------------------- |
| 数据库名     | `shiJiDB`                                                                                |
| 版本         | `1`                                                                                      |
| Object Store | `notes`                                                                                  |
| 主键         | `id`                                                                                     |
| 索引         | `category`（非唯一）、`tagColor`（非唯一）、`createdAt`（非唯一）、`updatedAt`（非唯一） |
### 3.3 存储策略
- **新增**：直接写入 IndexedDB
- **修改**：更新对应字段 + `updatedAt` 时间戳
- **删除（单条）**：软删除 → `isDeleted = true`，列表查询永远过滤 `isDeleted === true`
- **批量删除**：同上，逐条软删除
- **不实现硬删除**（基础版不提供回收站功能，软删除数据保留在本地但不展示，避免误删不可逆）
- **不提供导出功能**（基础版暂不实现，后续迭代可加）

## 四、功能需求详细说明
### 4.1 笔记列表页
#### 4.1.1 展示规则
- **排序**：按 `updatedAt` 降序（最近修改的在最前）
- **过滤**：`isDeleted === false` 且 `category` 匹配当前 Tab
- **每条笔记卡片展示**：
  - 第一行：标题（超长截断，最大显示 1 行，末尾省略号）
  - 第二行：正文前 ~60 个字符（超长截断，最大显示 2 行，末尾省略号）
  - 右侧：若有颜色标签，显示对应颜色的小圆点（直径 8px）
  - 底部辅助信息：相对时间（如「3 分钟前」「昨天 14:30」）
- **空状态**：分类下无笔记时，展示居中空状态插图 + 提示文案（各分类不同）
#### 4.1.2 空状态文案
| 分类                   | 文案                                   |
| ---------------------- | -------------------------------------- |
| 随想                   | 「还没有随想，点击右下角记录此刻」     |
| 学习                   | 「还没有学习笔记，点击右下角开始记录」 |
| 待办                   | 「还没有待办事项，点击右下角添加」     |
| 标签（某颜色无笔记）   | 「暂无此标签的笔记」                   |
| 标签（全部无标签笔记） | 「还没有被标记的笔记」                 |
#### 4.1.3 交互操作
| 操作         | 触发方式                                       | 行为                                |
| ------------ | ---------------------------------------------- | ----------------------------------- |
| 查看详情     | 点击笔记卡片                                   | 进入笔记详情页                      |
| 进入批量模式 | 长按任意笔记卡片（300ms）                      | 进入多选模式，当前卡片自动选中      |
| 批量选择     | 点击其他卡片                                   | 切换选中/取消选中状态               |
| 全选         | 批量模式顶部出现「全选」按钮                   | 选中当前列表所有笔记                |
| 批量删除     | 批量模式顶部出现「删除」按钮（选中数≥1时可用） | 弹出确认弹窗 → 确认后执行批量软删除 |
| 退出批量模式 | 点击顶部「取消」按钮 / 点击返回键              | 退出多选模式，清除所有选中状态      |
| 新建笔记     | 点击右下角 FAB 按钮（标签页隐藏 FAB）          | 进入新建笔记页                      |
#### 4.1.4 批量删除确认弹窗
- 文案：「确定删除 N 条笔记？」（N 为选中数量）
- 副文案：「删除后无法恢复」
- 按钮：「取消」（次要）、「删除」（危险色，如红色）
- 点击弹窗外部区域：关闭弹窗，不执行删除
### 4.2 标签页（二级导航）
#### 4.2.1 布局
- 顶部固定区域：水平排列 4 个颜色 Chip（红、橙、黄、灰），每个 Chip 内为色块圆点 + 文字标签名
- 默认状态：无 Chip 被选中，列表展示所有 `tagColor !== null` 的笔记
- 选中某颜色：列表筛选 `tagColor === 对应颜色` 的笔记，排序规则同 4.1.1
- Chip 选中态：白底 + 对应颜色边框 + 加深文字；未选中态：浅灰底
- **此页面无 FAB 按钮**（标签是自动归类视图，不允许直接新建）
#### 4.2.2 标签页中的笔记操作
- 点击笔记 → 进入详情页（同其他分类）
- 长按 → 进入批量模式（同其他分类）
- 在详情页中取消标签颜色 → 该笔记从标签视图中消失（因为是按 `tagColor !== null` 筛选的）
### 4.3 新建笔记页
#### 4.3.1 入口
- 在「随想」「学习」「待办」Tab 下，点击右下角 FAB 按钮
- `category` 自动设为当前 Tab 对应的值，用户不可更改
- `tagColor` 默认为 `null`
- `createdAt` 和 `updatedAt` 设为当前时间戳
#### 4.3.2 页面结构
- 顶部：返回按钮 + 标题「新建笔记」+ 占位（右侧空）
- 标题输入框：大字号、无下划线、placeholder「标题」
- 分隔线：细线 1px
- 正文输入区：占满剩余空间、自动聚焦、placeholder「开始记录...」
- 底部工具栏（固定）：左侧颜色标签选择区（4 个小圆点，点击切换，再次点击取消），右侧「保存」按钮
#### 4.3.3 保存逻辑
| 条件             | 行为                                   |
| ---------------- | -------------------------------------- |
| 标题和内容都为空 | 不保存，Toast 提示「请输入标题或内容」 |
| 仅有标题         | 允许保存                               |
| 仅有内容         | 允许保存（标题字段存空字符串）         |
| 标题或内容有值   | 保存成功 → Toast「已保存」→ 返回列表页 |
#### 4.3.4 自动保存（可选实现，建议实现）
- 用户输入时，每 3 秒自动保存一次（防丢失）
- 退出页面时（返回按钮 / 系统返回键），若存在未保存内容，自动保存
### 4.4 笔记详情页
#### 4.4.1 查看模式
- 顶部：返回按钮 + 标题「笔记详情」+ 右侧「编辑」图标按钮
- 展示标题（大字号、加粗）
- 展示正文（舒适行高 1.8、字号 15-16px）
- 底部：创建时间（小字、灰色）
#### 4.4.2 编辑模式
- 点击「编辑」按钮进入
- 顶部：返回按钮 + 标题「编辑笔记」+ 右侧「保存」按钮
- 标题和正文变为可编辑输入框（布局同新建页）
- 底部工具栏：颜色标签选择（显示当前标签状态）+ 「保存」按钮
- 保存逻辑：同 4.3.3，但更新 `updatedAt` 为当前时间戳
#### 4.4.3 删除操作
- 编辑模式下，顶部右侧「保存」按钮旁增加「删除」图标按钮
- 点击后弹出确认弹窗：「确定删除这条笔记？删除后无法恢复」
- 确认 → 软删除 → 返回列表页 → Toast「已删除」
#### 4.4.4 颜色标签操作细节
- 4 个颜色圆点水平排列
- 当前选中颜色：圆点外圈加白色环 + 略微放大
- 点击已选中的颜色圆点：取消标签（`tagColor` 设为 `null`）
- 点击未选中的颜色圆点：切换为该颜色（一个笔记同时只能有一个颜色标签）
- 切换/取消标签后：**立即生效并保存**，不需要点保存按钮（降低操作路径）
- Toast 反馈：「已标记为红色标签」/ 「已取消标签」
#### 4.4.5 系统返回键处理
- **查看模式**：返回 → 回到列表页
- **编辑模式且无修改**：返回 → 回到查看模式（不退出详情页）
- **编辑模式且有修改**：返回 → 弹出确认弹窗「有未保存的修改，是否保存？」→「不保存」回到查看模式 / 「保存」保存后留在查看模式
- **新建页面且有内容**：返回 → 弹出确认弹窗「是否保存草稿？」→「不保存」丢弃 / 「保存」保存后回到列表页
- **新建页面且无内容**：返回 → 直接返回列表页（不弹窗）

## 五、交互设计规范
### 5.1 触控区域
- 所有可点击元素的最小触控区域：**48 × 48 px**（符合 Material Design 和 WCAG 标准）
- 列表卡片整体可点击（非仅文字区域）
- 底部 Tab 图标触控区域：高度 56px
### 5.2 手势
| 手势       | 场景     | 行为                                 |
| ---------- | -------- | ------------------------------------ |
| 点击       | 笔记卡片 | 进入详情                             |
| 长按 300ms | 笔记卡片 | 进入批量选择模式                     |
| 点击       | FAB      | 新建笔记                             |
| 系统返回键 | 各页面   | 按 4.4.5 逻辑处理                    |
| 点击       | 底部 Tab | 切换分类（不刷新页面，局部更新列表） |
> **不实现左滑删除**（基础版不引入滑动手势，避免与系统返回手势冲突，且小米手机的边缘滑动返回是系统级行为）
### 5.3 动效规范
| 场景         | 动效                                      | 时长                            |
| ------------ | ----------------------------------------- | ------------------------------- |
| 页面进入     | 从右向左滑入                              | 250ms，ease-out                 |
| 页面返回     | 从左向右滑出                              | 200ms，ease-in                  |
| FAB 出现     | 从下方缩放弹入                            | 300ms，spring                   |
| 批量模式进入 | 顶部操作栏从上方滑入 + 卡片左侧出现勾选框 | 200ms                           |
| Toast        | 从底部滑入 + 淡出                         | 入 200ms，停留 2000ms，出 200ms |
| 弹窗         | 背景 fade in + 弹窗 scale from 0.95       | 200ms                           |
### 5.4 视觉风格定义
- **整体调性**：纯净、留白充足、无多余装饰、类似 Apple Notes / Google Keep 的克制感
- **背景色**：`#FAFAFA`（微暖灰白）
- **卡片色**：`#FFFFFF`
- **主文字**：`#1A1A1A`
- **次文字**：`#888888`
- **辅助文字**：`#BBBBBB`
- **分割线**：`#F0F0F0`
- **强调色**：`#1A1A1A`（黑，用于按钮和关键操作，不用彩色做主强调）
- **圆角**：卡片 12px，按钮 8px，弹窗 16px，FAB 16px
- **阴影**：卡片 `0 1px 3px rgba(0,0,0,0.04)`，极轻微，不抢眼
- **字体**：系统默认字体栈（`-apple-system, "PingFang SC", "Noto Sans SC", "Microsoft YaHei", sans-serif`），不引入外部字体（离线可用）
### 5.5 文字排版规范

## 六、边界情况与异常处理
### 6.1 输入边界
| 场景                    | 处理                                                                         |
| ----------------------- | ---------------------------------------------------------------------------- |
| 标题超过 100 字符       | 不阻止输入，但保存时截断为前 100 字符，Toast 提示「标题过长，已自动截断」    |
| 正文超过 50,000 字符    | 不阻止输入，但保存时截断为前 50,000 字符，Toast 提示「内容过长，已自动截断」 |
| 纯空白字符（空格/换行） | 视为空内容，触发「请输入标题或内容」提示                                     |
| 特殊字符（emoji 等）    | 正常保存和展示，不做限制                                                     |
### 6.2 数据边界
| 场景             | 处理                                                                   |
| ---------------- | ---------------------------------------------------------------------- |
| IndexedDB 不可用 | 启动时检测，若失败展示全屏错误页：「存储初始化失败，请检查浏览器设置」 |
| 存储空间不足     | 捕获 QuotaExceededError，Toast「存储空间不足」                         |
| 并发写入         | 单线程 JS 无此问题，无需处理                                           |
| 数据库版本升级   | 基础版固定版本号为 1，不做迁移逻辑                                     |
### 6.3 交互边界
| 场景                                     | 处理                                                        |
| ---------------------------------------- | ----------------------------------------------------------- |
| 批量模式下选中 0 条                      | 「删除」按钮置灰不可点击                                    |
| 批量模式下所有笔记被删完                 | 执行删除 → 显示空状态 → 自动退出批量模式                    |
| 快速连续点击保存                         | 防抖 500ms，避免重复写入                                    |
| 快速连续点击 FAB                         | 防抖 300ms                                                  |
| 编辑模式下标题和内容都被清空后点保存     | 视为空内容，Toast「请输入标题或内容」，不保存，留在编辑模式 |
| 新建时输入内容后不保存直接杀掉浏览器进程 | **数据丢失**（基础版可接受，自动保存可缓解）                |
| 从标签页进入笔记详情并取消标签 → 返回    | 笔记从当前标签列表中消失，列表刷新显示最新状态              |
| 从标签页进入笔记详情并修改了内容 → 返回  | 笔记的 `updatedAt` 更新，若列表按时间排序则位置可能变化     |
### 6.4 显示边界
| 场景                               | 处理                                                                                   |
| ---------------------------------- | -------------------------------------------------------------------------------------- |
| 标题为空字符串的笔记在列表中       | 标题行显示正文前 20 个字符作为占位标题，正文字段正常显示（可能看起来两行一样，可接受） |
| 正文为空字符串的笔记在列表中       | 摘要行不显示                                                                           |
| 标题和内容都为空（理论上不应存在） | 不展示，过滤掉                                                                         |
| 笔记正文超长（如 10000 字）        | 详情页正常滚动展示，不做折叠                                                           |
| 列表笔记数量极大（如 1000+）       | 使用虚拟滚动或分页加载（基础版建议超过 100 条时分页，每页 50 条，底部「加载更多」）    |
| 相对时间超过 7 天                  | 切换为绝对时间格式「2024-01-15 14:30」                                                 |
### 6.5 PWA 特有边界
| 场景                       | 处理                                                            |
| -------------------------- | --------------------------------------------------------------- |
| 离线状态下使用             | 全部功能正常（纯本地），无需特殊处理                            |
| 清除浏览器数据             | IndexedDB 被清空，数据丢失（不可逆，符合产品定位）              |
| Service Worker 更新        | 用户关闭所有标签页后重新打开即加载新版本                        |
| 添加到主屏幕后的状态栏颜色 | `theme_color` 设为 `#FAFAFA`，`background_color` 设为 `#FAFAFA` |

## 七、非功能需求
### 7.1 性能
- 列表页首屏渲染时间（FCP）：< 500ms
- 笔记详情页打开：< 200ms
- 保存操作：< 100ms（IndexedDB 写入）
- 列表滚动帧率：≥ 55fps
### 7.2 兼容性
- 最低要求：Chrome 80+（覆盖 2020 年后的安卓设备）
- 小米手机 MIUI 系统的 Chrome、Edge 浏览器均可
- 适配屏幕宽度：360px ~ 412px（主流安卓手机），不适配平板横屏
### 7.3 安全性
- 无网络请求，无 XSS 风险
- 用户输入纯文本，无 HTML 注入风险
- 不请求任何系统权限（不需要摄像头、定位、通知等）
### 7.4 可访问性
- 足够的文字对比度（主文字对比度 ≥ 4.5:1）
- 触控目标尺寸 ≥ 48px
- 不依赖颜色传递信息（颜色标签旁有文字标注）

## 九、页面布局详细规格
### 9.1 整体布局骨架
```
┌──────────────────────────┐  ← 状态栏区域（透明，PWA 全屏时隐藏）
│        顶部导航栏          │  ← 高度 56px，固定顶部
│  [←] 页面标题    [操作按钮] │
├──────────────────────────┤
│                          │
│                          │
│       内容区域            │  ← 可滚动，padding 16px
│       (列表/详情/编辑)     │
│                          │
│                          │
├──────────────────────────┤
│ [随想] [学习] [待办] [标签] │  ← 高度 56px + 安全区，固定底部
└──────────────────────────┘
```
### 9.2 底部 Tab 栏
- 高度：56px + 底部安全区 padding（`env(safe-area-inset-bottom)`）
- 背景：`#FFFFFF`，顶部 1px 分割线 `#F0F0F0`
- 每个Tab：图标（24px SVG）+ 文字（10px），垂直排列，间距 2px
- 选中态：图标和文字变为 `#1A1A1A`
- 未选中态：图标和文字变为 `#999999`
- 无角标、无红点
### 9.3 FAB 按钮
- 位置：右下角，距离底部 Tab 栏顶部 16px，距离右边 16px
- 尺寸：56 × 56px
- 圆角：16px
- 背景：`#1A1A1A`
- 图标：白色「+」号（24px）
- 阴影：`0 4px 12px rgba(0,0,0,0.15)`
- 点击态：缩放至 0.92，150ms
### 9.4 笔记卡片（列表项）
- 背景：`#FFFFFF`
- 圆角：12px
- 内边距：16px
- 卡片间距：8px
- 布局：
```
┌──────────────────────────────┐
│  标题文字（最多1行截断）   ●  │  ← ● 为颜色标签圆点（无标签时不显示）
│  正文摘要文字（最多2行截断）  │
│  3 分钟前                    │  ← 小字灰色
└──────────────────────────────┘
```
### 9.5 颜色标签选择器（编辑页/新建页底部）
- 4 个圆形色块，直径 24px，间距 16px，水平居中排列
- 未选中：对应颜色填充
- 选中：对应颜色填充 + 外圈 3px 白色环 + `box-shadow: 0 0 0 2px 对应颜色`
- 点击反馈：缩放至 0.85，100ms
### 9.6 标签页顶部 Chip 导航
- 水平排列，可横向滚动（若屏幕窄）
- 每个 Chip：高度 32px，圆角 16px（胶囊形），内边距 `0 12px`
- 内容：8px 色块圆点 + 4px 间距 + 12px 文字
- 未选中：背景 `#F5F5F5`，文字 `#666666`
- 选中：背景 `#FFFFFF`，2px 对应颜色边框，文字 `#1A1A1A`
- 间距：8px

## 十、相对时间格式化规则
| 时间差      | 显示                 |
| ----------- | -------------------- |
| < 1 分钟    | 「刚刚」             |
| 1 ~ 59 分钟 | 「N 分钟前」         |
| 1 ~ 23 小时 | 「N 小时前」         |
| 昨天        | 「昨天 HH:MM」       |
| 前天        | 「前天 HH:MM」       |
| 3 ~ 6 天    | 「N 天前」           |
| 7 天以上    | 「YYYY-MM-DD HH:MM」 |

## 十一、Toast 规范
- 位置：底部，距底部 Tab 栏上方 24px，水平居中
- 样式：圆角 8px，背景 `#1A1A1A`，文字白色 14px
- 内边距：`10px 20px`
- 无图标
- 无操作按钮
- 自动消失，不手动关闭
- 同一时间只展示一条，新的替换旧的
---
## 十二、确认弹窗规范
- 遮罩层：`rgba(0,0,0,0.4)`，点击遮罩关闭弹窗
- 弹窗：宽度 `min(280px, 80vw)`，背景白色，圆角 16px
- 内边距：24px
- 标题：居中，17px，加粗，底部间距 8px
- 正文：居中，14px，灰色，底部间距 24px
- 按钮区：两个按钮水平排列，等宽，高度 40px，圆角 8px
- 取消按钮：背景 `#F5F5F5`，文字 `#666666`
- 确认按钮（普通）：背景 `#1A1A1A`，文字白色
- 确认按钮（危险）：背景 `#E54D42`，文字白色
- 按钮间距：12px
